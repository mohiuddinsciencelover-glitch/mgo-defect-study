01 / Theory

The Master Formula

Every defect formation energy in this study is computed from this single equation. Understanding each term — and its sign convention — is essential before running a single VASP calculation.

Ef[Xq]  =  Etot[Xq]  −  Etot[host]  −  Σi niμi  +  q(EVBM + ΔEF)  +  Ecorr
Formation energy of defect X in charge state q at Fermi level position ΔEF relative to the VBM
Etot[Xq]
Total DFT energy of the defect supercell in charge state q From VASP OUTCAR → energy(sigma→0). Use ISMEAR=-5 for final static run.
Etot[host]
Total DFT energy of the perfect bulk supercell (same size, no defect) Must use identical ENCUT, k-mesh, ISIF=2, and cell vectors as the defect cell.
Σ niμi
Chemical potential sum: energy cost/gain from exchanging atoms with external reservoir ni = +1 when atom added (interstitial), −1 when removed (vacancy). This is the most common source of sign errors.
q(EVBM + ΔEF)
Electron chemical potential: energy of electrons exchanged with the Fermi reservoir EVBM from bulk OUTCAR. Scan ΔEF from 0 (VBM) to Egap (CBM) to get the full formation energy diagram.
Ecorr
Finite-size correction (Freysoldt/FNV) Mandatory for q ≠ 0. Removes spurious electrostatic interaction between periodic images of the charged defect. Use pydefect after all runs.
⚠
Critical sign convention: ni is positive when an atom is ADDED to the supercell and negative when REMOVED. The formula term is −niμi, so for a vacancy (n = −1) this becomes +μi. Getting this backwards is the single most common error in defect calculations.

Sign Convention by Defect Type

Defect Type Operation ni Formula term −niμi
Vacancy (V_Mg) Remove 1 Mg from supercell n = −1 +μMg
Interstitial (Mgi) Add 1 Mg to supercell n = +1 −μMg
Antisite (MgO) Remove 1 O, add 1 Mg nO = −1 nMg = +1 +μO − μMg
Frenkel pair (V_Mg + Mgi) Remove 1, add 1 (net zero) n = 0 net Ef = Ef[VMg] + Ef[Mgi]
Schottky (V_Mg + V_O) Remove 1 Mg + 1 O nMg = −1 nO = −1 +μMg + μO

05 / Workflow

Execution Order

The calculations must be run in this order. Each step depends on the previous one. The automation script handles this dependency chain automatically.

STEP 01
Convergence Tests (primitive cell)
Sweep ENCUT (400–600 eV) and k-mesh (4x4x4 to 12x12x12) on the 2-atom primitive cell. Accept when ΔE < 1 meV/atom. These values lock in for every subsequent calculation.
STEP 02
Bulk MgO Relaxation (ISIF=3)
Full ionic + cell relaxation of the 64-atom 2×2×2 supercell. Obtain equilibrium lattice parameter. Save WAVECAR and CONTCAR. This CONTCAR becomes the starting geometry for ALL defect POSCARs.
STEP 03
Bulk MgO Static (NSW=0)
Single-point calculation with ISMEAR=−5 on the relaxed geometry. Extract Etot[host] and EVBM from OUTCAR. This is your reference energy for every defect formation energy.
STEP 04
Elemental References (parallel)
Mg HCP bulk (ISMEAR=1, Methfessel-Paxton) and O2 molecule (ISPIN=2, triplet state, 15 Å vacuum box). Apply GGA overbinding correction: μO = E(O2)/2 − 0.36 eV.
STEP 05
Chemical Potential Limits
Compute ΔHf(MgO) = EMgO/f.u. − μ0Mg − μ0O. Determine Mg-rich (ΔμMg=0) and O-rich (ΔμO=0) limits.
STEP 06
Build Defect POSCARs
Use pymatgen to create all defect supercells from the bulk CONTCAR. Vacancies (63 atoms), interstitials (65 atoms at octahedral void), antisites (64 atoms, swapped species). The automation script does this automatically.
STEP 07
Defect Relaxations (parallel, ISIF=2)
All 38 defect relaxations run in parallel. ISIF=2 is mandatory — cell vectors must remain identical to the bulk reference. ISPIN=2 for all defects initially; check magnetisation and rerun with ISPIN=1 if ~0.
STEP 08
Defect Static Runs
Single-point calculations on relaxed defect geometries using ISMEAR=−5. WAVECAR from relaxation seeds the SCF. Extract final Etot[Xq] from each OUTCAR.
STEP 09
FNV Finite-Size Corrections
Apply Freysoldt/FNV corrections for all charged defects (q ≠ 0) using pydefect. This requires the dielectric tensor from a separate DFPT run. Corrections are typically 0.1–0.5 eV for q=±1 in a 64-atom cell.
STEP 10
Post-Processing and Plots
Collect all total energies. Compute Ef[Xq] for each defect and charge state as a function of ΔEF. Plot formation energy diagrams, charge transition level diagrams, and chemical potential stability triangle.

06 / HPC Setup

Running on Your Own HPC Cluster

This guide applies to any Linux HPC cluster with VASP installed. No specific server information is included — adapt paths and module names to your system.

1

Prerequisites

You need: VASP 5.4+ or 6.x with a valid licence, PAW PBE pseudopotentials (POTCAR files), vaspkit for automated POTCAR generation, and MPI (OpenMPI or IntelMPI).

2

Python Environment

Create a conda environment with Python 3.11, pymatgen, ASE, matplotlib, and tabulate. If your cluster has no internet, use a local conda mirror or pip with cached wheels.

3

NCORE Setting

Set NCORE = sqrt(total_cores). For 40 cores: NCORE=6. For 64 cores: NCORE=8. This controls VASP's internal parallelisation and has a large impact on performance.

4

Job Scheduler

If your cluster uses SLURM: sbatch submit.sh. PBS/Torque: qsub submit.sh. LSF: bsub < submit.sh. If no scheduler (standalone), run mpirun directly. The automation script auto-detects or runs directly.

5

POTCAR Generation

Use vaspkit option 103 in each calculation directory. It reads your POSCAR and selects the correct PAW PBE potential automatically. Verify ENMAX in the POTCAR matches your expected value (O: ~400 eV).

6

Verify NELECT

After the first neutral run, check: grep "NELECT" OUTCAR | head -1. Standard PAW PBE: Mg=2e, O=6e, giving 256 for a 64-atom cell. If using Mg_pv (10e), recalculate all NELECT values for charged defects.

✓
Automation script available: A Python script (run_workflow.py) automates the entire workflow — building POSCARs from the CIF, writing all INCARs and KPOINTS, generating POTCARs via vaspkit, launching all 44 calculations with correct dependency ordering (bulk before defects, relax before static), monitoring progress every 30 minutes, and running post-processing automatically. Works on any Linux cluster with or without a job scheduler.

Quick Start Commands

bash — on your HPC cluster 
# 1. Create the Python environment (login node, with internet)
conda create -n mgo_defects python=3.11 numpy scipy matplotlib pandas -y
conda activate mgo_defects
pip install pymatgen ase tabulate

# 2. Test dry-run (builds all files, no VASP launched)
python3 run_workflow.py --dry-run

# 3. Build all input files
python3 run_workflow.py --only-setup

# 4. Generate POTCARs using vaspkit
bash MgO_defect_study/generate_all_potcars.sh

# 5. Launch all 44 calculations (background, survives SSH disconnect)
nohup python3 run_workflow.py > workflow_master.log 2>&1 &

# 6. Watch progress
tail -f workflow_master.log

# 7. After completion: run analysis and generate plots
python3 run_workflow.py --only-analysis
⚠
Finite-size corrections are not automated. After all charged defect calculations finish, apply FNV corrections manually using pydefect. This requires a separate DFPT calculation to obtain the dielectric tensor (ε0). Corrections are typically 0.1–0.5 eV for |q|=1 in a 64-atom cell and can be as large as 1–2 eV for |q|=2.
